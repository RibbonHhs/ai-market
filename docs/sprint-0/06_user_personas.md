# User Personas（用户角色卡）

> 作者：pm-alice @ Sprint 0 Kickoff (2026-06-06)
>
> 版本：v1.0 · 范围：v1 · 验证状态：基于 PRD 假设（Sprint 2 补 3 场访谈验证）· 引用：`docs/PRD.md` §2 目标用户

## 0. 方法与边界

- 提炼自 `PRD.md` §2 + `ER.md` 角色字段（USER.role: ADMIN/USER）
- 共 3 类（与 PRD 对齐：AI 开发者 / Skill 创作者 / 运营 Admin）
- 每卡含：人口学 / 目标 / 痛点 / 行为 / 触点 / 关键 KPI / 反例
- 验证清单：每类 Persona 在 Sprint 2 内补 1 场访谈（共 3 场），由 pm-alice 跟进

## 1. Persona A — 开发者"陈昊"（AI Developer / 浏览者）

```
┌────────────────────────────────────────────────────────┐
│  姓名：陈昊                                             │
│  年龄：28                                               │
│  职业：全栈工程师（中型 SaaS 公司）                       │
│  技术栈：Claude Code / Cursor / VS Code                │
│  工作场景：每天 4–6 小时在 IDE 中编码                    │
│  设备：MacBook Pro 14" M3 + 双显示器                    │
│  网速：高速                                              │
│  经验：3 年 web 后端，1 年 AI 辅助编程                   │
└────────────────────────────────────────────────────────┘
```

### 1.1 画像与目标

| 维度 | 内容 |
|---|---|
| 身份 | 浏览者（USER 角色，未注册也可浏览大部分内容） |
| 痛点 | 在 Anthropic 官方 plugin 市场找 Skill 像大海捞针；中文社区资料少 |
| 目标 | 5 分钟内找到适合当前任务的 Skill，复制命令后立即使用 |
| 行为 | Google 搜索 "claude skill xxx" → 进 skillsmap.com → 关键词搜 → 看评分 → 安装 |

### 1.2 用户旅程（典型 1 次）

1. 打开首页（搜索引擎 / 直接输入）
2. Hero 搜索框输入 "test"
3. 看搜索结果列表，扫一眼标题 + 评分
4. 点进详情页，看 SKILL.md + 评分列表
5. 点复制安装命令 → 粘到终端 → 完事

### 1.3 触点（必须好用）

| 触点 | 期望体验 | 关键 KPI |
|---|---|---|
| 首页加载 | ≤ 1.5s | LCP |
| 搜索响应 | ≤ 300ms（首屏） | TTFB |
| 详情页 Markdown | 排版清爽 + 代码高亮 + 目录锚点 | 跳出率 |
| 复制按钮 | 一击即中 + toast 反馈 | 点击成功率 |
| 未登录评价 | 引导"去登录"而非阻塞 | 注册转化 |

### 1.4 反例（不该发生）

- ❌ 详情页加载 5s+ 还在转圈
- ❌ 搜索结果 0 个时显示空白
- ❌ 复制按钮藏在小图标里，要 hover 0.5s 才显示
- ❌ SKILL.md 显示为纯文本没排版

## 2. Persona B — 创作者"李雪"（Skill Creator / 贡献者）

```
┌────────────────────────────────────────────────────────┐
│  姓名：李雪                                             │
│  年龄：32                                               │
│  职业：独立开发者 / 自由职业                               │
│  技术栈：Claude API + Python + Markdown                │
│  工作场景：自己写 Skill 并希望被更多人使用                  │
│  设备：MacBook Air + iPad                                │
│  网速：中等                                              │
│  经验：写过 3 个 Skill，最受欢迎的被装 200+ 次             │
└────────────────────────────────────────────────────────┘
```

### 2.1 画像与目标

| 维度 | 内容 |
|---|---|
| 身份 | 贡献者（USER 角色 + 后台权限 / 或 仅 USER 走贡献渠道） |
| 痛点 | 自己的 Skill 没有曝光渠道；后台发布流程繁琐；看不到下载量 |
| 目标 | 把写好的 Skill 推上平台，看下载 / 评分反馈 |
| 行为 | 在本地写 SKILL.md → 进 `/admin/skills/new` → 上传单个 .md 或 .skill 包 → 提交 |

### 2.2 用户旅程（典型 1 次）

1. 在 Cursor 写好 SKILL.md（含 frontmatter）
2. 进 SkillsMap 后台 → Skill 管理 → 新建
3. 选"单个 .md" → 拖拽文件 → 系统自动解析 frontmatter 预填
4. 补全剩余字段（分类、标签）
5. 提交 → 状态默认 draft
6. 点"发布" → 立即在 `/browse` 看到

### 2.3 触点（必须好用）

| 触点 | 期望体验 | 关键 KPI |
|---|---|---|
| SKILL.md 上传解析 | 拖入即自动填表单 | 解析成功率 |
| 字段必填提示 | 明确不模糊 | 表单放弃率 |
| 发布后即时可见 | 提交 ≤ 2s 出现在 `/browse` | 时延 |
| 我的 Skill 列表 | 看到 installs / views / rating_avg | 数据完整性 |
| 编辑 / 上下架 | 单步完成 | 操作步骤数 |

### 2.4 反例

- ❌ 上传 SKILL.md 后表单还是空的（解析失败）
- ❌ 发布后 5 分钟 `/browse` 还看不到
- ❌ 找不到"我的 Skill"入口
- ❌ 删除 Skill 时没 confirm 直接清空

## 3. Persona C — 运营"周明"（Platform Admin / 管理员）

```
┌────────────────────────────────────────────────────────┐
│  姓名：周明                                             │
│  年龄：35                                               │
│  职业：SkillsMap 平台运营 / SRE                           │
│  工作场景：白天处理内容审核，晚上偶尔处理线上告警           │
│  设备：Windows 11 + 公司配 4K 显示器                      │
│  网速：公司内网                                          │
│  经验：8 年互联网运营，熟悉 SQL 与基础 Linux                │
└────────────────────────────────────────────────────────┘
```

### 3.1 画像与目标

| 维度 | 内容 |
|---|---|
| 身份 | 管理员（ADMIN 角色） |
| 痛点 | 每天要处理 10+ 个新 Skill 审核；分类 / 标签容易乱；用户违规难追责 |
| 目标 | 高效审核 + 维护分类 / 标签整洁 + 风险账号及时处理 |
| 行为 | 早晨看 Dashboard → 审新 Skill → 处理举报 → 整理分类 / 标签 |

### 3.2 用户旅程（典型 1 天）

1. 登录 `/admin` → 看 Dashboard 数字
2. 进 `/admin/skills` → 状态过滤 "draft" → 逐个审核
3. 没问题点"发布"；有问题点"编辑"或"下架"
4. 顺手处理举报：在 `/admin/users` 找到当事人 → 改角色 / 启禁
5. 整理标签：合并同义 / 删除空 tag
6. 每周一次"从本地扫描"导入新 Skill

### 3.3 触点（必须好用）

| 触点 | 期望体验 | 关键 KPI |
|---|---|---|
| 登录 | admin/admin123 默认账号可登；首登强提示改密码 | 登录成功率 |
| Dashboard | 加载 ≤ 1.5s，数据有"更新时间" | 数据新鲜度 |
| Skill 列表过滤 | 状态 / 分类 / 来源多维过滤 | 操作次数 |
| 危险操作 | 删除 / 启禁 / 改角色 confirm 弹窗 | 误操作率 |
| 本地扫描 | 一键触发，进度可视，完成有报告 | 完成度 |

### 3.4 反例

- ❌ 删 Skill 没二次确认
- ❌ 启禁后用户不知情
- ❌ 标签删除误删带数据的
- ❌ 本地扫描一次要 5 分钟没进度条

## 4. 3 类 Persona 对照

| 维度 | A 开发者"陈昊" | B 创作者"李雪" | C 运营"周明" |
|---|---|---|---|
| 角色 | USER（未登录即可） | USER（注册即可） | ADMIN |
| 主要场景 | 浏览 + 决策 + 评分 | 上传 + 管理自己 | 审核 + 维护平台 |
| 频率 | 偶尔（按需） | 每周 1-2 次 | 每天 |
| 关键页 | `/` `/browse` `/skills/:id` | `/admin/skills/new` | `/admin/*` |
| 关键 KPI | 搜索成功率 | 上架成功率 | 审核 throughput |
| 设备 | Mac 高分辨率 | Mac / iPad | Windows 4K |

## 5. 跨 Persona 通用需求

- **可访问性**（A11y）：键盘可导航、屏幕阅读器友好
- **性能**：90% 页面 LCP ≤ 1.5s
- **安全**：登录 token 加密存储 + 角色严格校验
- **国际化**：中英双语（Could 优先级）
- **响应式**：在 1024-1920 宽度下显示正确（后台除外，可只支持 ≥ 1280）
